## 1. 产品概述
### 1.1 产品愿景
菜谱分享系统是一个基于Web的智能美食平台，致力于为用户提供个性化的菜谱发现、学习和分享体验。通过智能化推荐和社区互动，让每个人都能轻松找到适合自己的健康美味菜谱。

### 1.2 产品定位
- **目标市场**: 本地化美食社区平台
- **核心价值**: 智能推荐 + 社区分享 + 健康饮食
- **差异化优势**: 个性化推荐算法 + 高质量内容社区 + 本地化服务

### 1.3 目标用户

#### 主要用户群体：都市健康生活家
- **年龄**: 25-45岁
- **特征**: 中高收入、本科以上学历、一二线城市
- **行为**: 每周在家做饭3-5次，关注健康饮食
- **痛点**: 缺乏系统营养知识、需要高效菜谱解决方案

#### 次要用户群体：专业厨师和美食博主
- **特征**: 有专业烹饪背景，希望建立个人品牌
- **需求**: 专业的作品展示和管理工具

### 1.4 商业目标
- **短期目标** (6个月): 1000名活跃用户，月留存率40%
- **中期目标** (12个月): 5000名活跃用户，实现初步营收
- **长期目标**: 成为本地最具影响力的智能美食社区

## 2. 用户故事和用例

### 2.1 核心用户故事

#### 用户注册与认证
- **作为一个** 新用户
- **我希望** 能够快速注册账户
- **以便** 开始使用平台的菜谱功能

**验收标准:**
- 支持邮箱注册
- 支持手机号注册
- 注册流程不超过3步
- 支持邮箱验证

#### 菜谱浏览与搜索
- **作为一个** 用户
- **我希望** 能够搜索感兴趣的菜谱
- **以便** 找到适合自己口味和需求的菜品

**验收标准:**
- 支持按菜名搜索
- 支持按食材搜索
- 搜索结果相关性高
- 响应时间<2秒

#### 菜谱创建与分享
- **作为一个** 美食爱好者
- **我希望** 能够创建和分享自己的菜谱
- **以便** 与社区交流烹饪经验

**验收标准:**
- 支持菜谱基本信息录入
- 支持图片上传
- 支持步骤说明
- 支持标签分类

#### 个人中心管理
- **作为一个** 注册用户
- **我希望** 能够管理个人信息和收藏
- **以便** 个性化使用体验

**验收标准:**
- 个人信息编辑
- 收藏夹管理
- 历史记录查看
- 偏好设置

### 2.2 详细用例

#### 用例1: 用户注册
**参与者**: 新用户
**前置条件**: 无
**基本流程**:
1. 用户访问注册页面
2. 填写邮箱/手机号、密码、用户名
3. 同意用户协议和隐私政策
4. 提交注册信息
5. 系统发送验证邮件/短信
6. 用户完成验证
7. 注册成功，自动登录

#### 用例2: 菜谱搜索
**参与者**: 注册用户/访客
**前置条件**: 无
**基本流程**:
1. 用户在搜索框输入关键词
2. 选择搜索类型（菜名/食材）
3. 系统返回匹配结果
4. 用户可按条件筛选（难度、时长等）
5. 用户点击查看菜谱详情

#### 用例3: 菜谱创建
**参与者**: 注册用户
**前置条件**: 用户已登录
**基本流程**:
1. 用户点击"创建菜谱"
2. 填写菜谱基本信息（名称、简介、难度等）
3. 上传菜谱图片
4. 添加食材清单和用量
5. 编写制作步骤
6. 添加标签和分类
7. 预览并发布菜谱

## 3. 功能需求详细说明

### 3.1 用户认证系统

#### 3.1.1 用户注册
- **功能描述**: 新用户创建账户
- **输入字段**: 邮箱/手机号、密码、确认密码、用户名
- **验证规则**:
  - 邮箱格式验证
  - 手机号格式验证
  - 密码强度要求（8-20位，包含字母数字）
  - 用户名唯一性检查
- **业务规则**:
  - 同一邮箱/手机号只能注册一次
  - 需要通过邮箱/短信验证
  - 用户同意服务条款才能注册

#### 3.1.2 用户登录
- **功能描述**: 已注册用户身份验证
- **登录方式**: 邮箱+密码、手机号+密码
- **安全特性**:
  - 密码加密存储
  - 登录失败次数限制（5次后锁定30分钟）
  - 记住登录状态（可选）
  - 自动登出时间（7天）

#### 3.1.3 密码重置
- **功能描述**: 用户忘记密码时重置
- **流程**:
  1. 输入注册邮箱/手机号
  2. 系统发送重置链接/验证码
  3. 用户验证身份
  4. 设置新密码

### 3.2 菜谱管理系统

#### 3.2.1 菜谱创建 (Create)
- **功能描述**: 用户创建新菜谱
- **必填字段**:
  - 菜谱名称（1-50字符）
  - 菜谱简介（10-500字符）
  - 难度等级（简单/中等/困难）
  - 制作时间（分钟）
  - 食材清单（至少1种）
  - 制作步骤（至少1步）
- **选填字段**:
  - 菜谱图片（最多5张，每张<5MB）
  - 营养信息（热量、蛋白质等）
  - 小贴士
  - 标签（最多5个）
- **业务规则**:
  - 菜草稿可保存
  - 发布后可编辑
  - 删除需要确认

#### 3.2.2 菜谱浏览 (Read)
- **功能描述**: 用户查看菜谱列表和详情
- **列表展示**:
  - 菜谱封面图
  - 菜谱名称
  - 作者信息
  - 评分和评论数
  - 制作时间
  - 难度等级
- **详情页面**:
  - 完整菜谱信息
  - 高清图片展示
  - 分步骤制作说明
  - 营养信息（如有）
  - 作者其他菜谱推荐

#### 3.2.3 菜谱编辑 (Update)
- **功能描述**: 菜谱作者修改已发布菜谱
- **权限**: 仅菜谱创建者可编辑
- **编辑限制**:
  - 已有评论的菜谱，核心内容修改需要标记
  - 修改记录可追溯
- **版本控制**: 保留最近3个版本

#### 3.2.4 菜谱删除 (Delete)
- **功能描述**: 菜谱作者删除菜谱
- **权限**: 仅菜谱创建者可删除
- **删除策略**:
  - 软删除（标记为已删除）
  - 30天内可恢复
  - 超过30天永久删除
- **影响处理**:
  - 相关评论一并隐藏
  - 用户收藏自动移除

### 3.3 图片上传管理

#### 3.3.1 图片上传
- **支持格式**: JPG, PNG, GIF
- **大小限制**: 单张图片<5MB
- **尺寸要求**:
  - 最小尺寸：400x300
  - 推荐尺寸：1200x800
- **上传方式**:
  - 本地文件选择
  - 拖拽上传
- **图片处理**:
  - 自动压缩优化
  - 生成缩略图
  - 添加水印（可选）

#### 3.3.2 图片管理
- **功能**: 查看、删除、替换图片
- **存储**: 本地文件系统
- **备份**: 定期备份到云存储

### 3.4 搜索功能

#### 3.4.1 基础搜索
- **搜索范围**: 菜谱名称、食材、简介
- **搜索方式**:
  - 关键词搜索
  - 模糊匹配
  - 拼音搜索（支持中文）
- **搜索建议**:
  - 热门搜索词推荐
  - 搜索历史记录
  - 自动补全

#### 3.4.2 高级筛选
- **筛选条件**:
  - 菜系分类（川菜、粤菜、西餐等）
  - 难度等级
  - 制作时间
  - 热门程度
  - 发布时间
- **排序选项**:
  - 相关度（默认）
  - 最新发布
  - 评分最高
  - 评论最多

### 3.5 用户个人中心

#### 3.5.1 个人信息管理
- **可编辑信息**:
  - 头像上传
  - 昵称
  - 个人简介
  - 地区
  - 烹饪偏好
- **隐私设置**:
  - 菜谱可见性
  - 个人信息公开程度
  - 消息通知设置

#### 3.5.2 收藏夹管理
- **功能**:
  - 收藏菜谱
  - 创建收藏分类
  - 管理收藏列表
  - 批量操作
- **收藏限制**:
  - 单个用户最多收藏1000个菜谱
  - 支持导出收藏列表

#### 3.5.3 历史记录
- **浏览历史**: 最近查看的菜谱
- **搜索历史**: 搜索关键词记录
- **操作历史**: 发布、编辑、评论记录

### 3.6 评论评分系统

#### 3.6.1 评分功能
- **评分方式**: 5星评分制
- **评分规则**:
  - 每个用户对同一菜谱只能评分一次
  - 评分可修改（最近一次评分生效）
  - 平均评分实时计算

#### 3.6.2 评论功能
- **评论内容**:
  - 文字评论（10-500字符）
  - 图片评论（可选，最多3张）
- **评论管理**:
  - 发表评论
  - 编辑自己的评论
  - 删除自己的评论
  - 举报不当评论

#### 3.6.3 互动功能
- **点赞**: 对评论或菜谱点赞
- **回复**: 评论之间的回复互动
- **通知**: 评论被回复时通知

## 5. 技术规范

### 5.1 技术架构

#### 5.1.1 整体架构
```
前端 (Vue.js) ←→ 后端API (Flask) ←→ 数据库 (MySQL)
                ↓
            缓存层 (Redis)
                ↓
            文件存储 (本地文件系统)
```

#### 5.1.2 前端技术栈
- **框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **UI组件**: Element Plus
- **状态管理**: Pinia
- **路由管理**: Vue Router
- **HTTP客户端**: Axios
- **样式**: SCSS + CSS Modules

#### 5.1.3 后端技术栈
- **框架**: Flask + Flask-RESTful
- **数据库ORM**: SQLAlchemy
- **认证**: Flask-JWT-Extended
- **数据库**: MySQL 5.7.44
- **缓存**: Redis
- **任务队列**: Celery
- **文件上传**: Flask-Uploads

#### 5.1.4 开发工具
- **版本控制**: Git
- **代码规范**: ESLint, Prettier, Black
- **API文档**: Flask-RESTX (Swagger)
- **测试**: Jest (前端), pytest (后端)

### 5.2 数据库设计

#### 5.2.1 核心表结构

**用户表 (users)**
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    avatar VARCHAR(255),
    bio TEXT,
    location VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE,
    email_verified BOOLEAN DEFAULT FALSE
);
```

**菜谱表 (recipes)**
```sql
CREATE TABLE recipes (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(100) NOT NULL,
    description TEXT,
    difficulty ENUM('easy', 'medium', 'hard') NOT NULL,
    prep_time INT NOT NULL,
    cook_time INT,
    servings INT,
    author_id INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_published BOOLEAN DEFAULT FALSE,
    is_deleted BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (author_id) REFERENCES users(id)
);
```

**食材表 (ingredients)**
```sql
CREATE TABLE ingredients (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    unit VARCHAR(20),
    category VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**菜谱食材关联表 (recipe_ingredients)**
```sql
CREATE TABLE recipe_ingredients (
    id INT PRIMARY KEY AUTO_INCREMENT,
    recipe_id INT NOT NULL,
    ingredient_id INT NOT NULL,
    amount DECIMAL(10,2),
    notes VARCHAR(255),
    FOREIGN KEY (recipe_id) REFERENCES recipes(id),
    FOREIGN KEY (ingredient_id) REFERENCES ingredients(id)
);
```

**制作步骤表 (recipe_steps)**
```sql
CREATE TABLE recipe_steps (
    id INT PRIMARY KEY AUTO_INCREMENT,
    recipe_id INT NOT NULL,
    step_number INT NOT NULL,
    description TEXT NOT NULL,
    image VARCHAR(255),
    FOREIGN KEY (recipe_id) REFERENCES recipes(id)
);
```

**评论表 (comments)**
```sql
CREATE TABLE comments (
    id INT PRIMARY KEY AUTO_INCREMENT,
    recipe_id INT NOT NULL,
    user_id INT NOT NULL,
    content TEXT NOT NULL,
    rating INT CHECK (rating >= 1 AND rating <= 5),
    parent_id INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    is_deleted BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (recipe_id) REFERENCES recipes(id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (parent_id) REFERENCES comments(id)
);
```

**收藏表 (favorites)**
```sql
CREATE TABLE favorites (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    recipe_id INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (recipe_id) REFERENCES recipes(id),
    UNIQUE KEY unique_favorite (user_id, recipe_id)
);
```

- **资源导向**: 使用名词表示资源

- **版本控制**: URL版本控制 (/api/v1/)

```json
  "data": {},
  "message": "操作成功",
  "timestamp": "2025-12-11T10:00:00Z",
  "code": 200
}
```

#### 5.3.3 分页格式
```json
{
  "success": true,
  "data": {
    "items": [],
    "pagination": {
      "page": 1,
      "per_page": 20,
      "total": 100,
      "pages": 5
    }
  }
}
```
