```vue
<template>
  <div class="admin-users">
    <div class="page-header">
      <h2>用户管理</h2>

    </div>

          <template #default="scope">

              {{ scope.row.is_admin ? '是' : '否' }}
            </el-tag>
          </template>
        </el-table-column>
      </el-table>

      <!-- 分页 -->
      <el-pagination
        v-if="pagination.total > 0"
        @current-change="handlePageChange"
        :current-page="pagination.currentPage"
        :page-size="pagination.pageSize"
        :total="pagination.total"
        layout="total, prev, pager, next"
        style="margin-top: 20px; justify-content: center"
      />
    </el-card>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { ElMessage } from 'element-plus'
import api from '@/api/index'

const loading = ref(false)
const users = ref([])
const pagination = ref({
  currentPage: 1,
  pageSize: 10,
  total: 0
})

// 获取用户列表
const fetchUsers = async (page = 1) => {
  loading.value = true
  try {
    const data = await api.get('/admin/users', {
      params: {
        page: page,
        per_page: pagination.value.pageSize
      }
    })

    if (data) {
      users.value = data.users
      pagination.value.total = data.total
      pagination.value.currentPage = data.current_page
    }
  } catch (error: any) {
    console.error('获取用户列表失败:', error)
    ElMessage.error('获取用户列表失败')
  } finally {
    loading.value = false
  }
}

// 处理分页
const handlePageChange = (page: number) => {
  fetchUsers(page)
}

// 格式化日期
const formatDate = (dateString: string) => {
  const date = new Date(dateString)
  return date.toLocaleString('zh-CN')
}

onMounted(() => {
  fetchUsers()
})
</script>
```

**功能特点**：
- **表格展示**：清晰的列表视图
- **分页加载**：支持大量用户数据

```python
@admin_bp.route('/statistics', methods=['GET'], endpoint='admin_statistics')
def admin_get_statistics():
    """获取系统统计数据"""
    try:
        # 1. 验证JWT Token
        verify_jwt_in_request()

        # 2. 统计用户数量
        user_count = db.session.query(User).count()

        # 3. 统计菜谱数量
        recipe_count = db.session.query(Recipe).count()

        # 4. 统计评论数量
        comment_count = db.session.query(Comment).count()

        # 5. 统计收藏数量
        favorite_count = db.session.query(Favorite).count()

        # 6. 统计点赞数量
        like_count = db.session.query(Like).count()

        # 7. 返回统计数据
        return jsonify({
            'user_count': user_count,
            'recipe_count': recipe_count,
            'comment_count': comment_count,
            'favorite_count': favorite_count,
            'like_count': like_count
        })
    except Exception as e:
        # 判断是否是JWT相关错误
        if 'authorization' in str(e).lower() or 'token' in str(e).lower():
            return jsonify({'message': '请先登录'}), 401
        return jsonify({'message': f'获取统计数据失败: {str(e)}'}), 500
```

**技术要点**：
- **JWT验证**：`verify_jwt_in_request()` 确保已登录
- **数据库聚合查询**：使用 `count()` 快速统计
- **错误分类处理**：区分JWT错误和其他错误
- **JSON响应**：标准化数据格式

```python
@admin_bp.route('/users', methods=['GET'], endpoint='admin_users')
def admin_get_users():
    """获取所有用户列表"""
    try:
        # 1. 验证JWT Token
        verify_jwt_in_request()

        # 2. 获取分页参数
        page = request.args.get('page', 1, type=int)
        per_page = request.args.get('per_page', 10, type=int)

        # 3. 分页查询用户
        users = db.session.query(User).paginate(
            page=page,
            per_page=per_page,
            error_out=False
        )

        # 4. 返回用户列表和分页信息
        return jsonify({
            'users': [user.to_dict() for user in users.items],
            'total': users.total,
            'pages': users.pages,
            'current_page': page
        })
    except Exception as e:
        # 判断是否是JWT相关错误
        if 'authorization' in str(e).lower() or 'token' in str(e).lower():
            return jsonify({'message': '请先登录'}), 401
        return jsonify({'message': f'获取用户列表失败: {str(e)}'}), 500
```

**核心逻辑**：
1. **身份验证**：验证管理员身份
2. **分页查询**：使用 `paginate()` 方法
3. **数据转换**：调用 `to_dict()` 转换为JSON
4. **分页信息**：返回总数、总页数、当前页

### 8. 前端路由权限控制

**文件位置**：`frontend/src/router/index.ts`

```typescript
{
  path: '/admin',
  component: () => import('@/views/admin/AdminLayout.vue'),
  meta: { requiresAuth: true, requiresAdmin: true },  // ⭐ 关键：需要管理员权限
  children: [
    {
      path: 'dashboard',
      component: () => import('@/views/admin/Dashboard.vue')
    },
    {
      path: 'users',
      component: () => import('@/views/admin/Users.vue')
    },
    {
      path: 'recipes',
      component: () => import('@/views/admin/Recipes.vue')
    },
    {
      path: 'comments',
      component: () => import('@/views/admin/Comments.vue')
    }
  }
}
```

**路由守卫实现**：

```typescript
router.beforeEach((to, from, next) => {
  const isAuthenticated = authApi.isLoggedIn()

  // 检查是否需要登录
  if (to.meta.requiresAuth && !isAuthenticated) {
    next('/login')
    return
  }

  // 检查是否需要管理员权限
  if (to.meta.requiresAdmin) {
    const user = authApi.getCurrentUser()
    if (!user || !user.is_admin) {
      // 不是管理员，跳转回首页
      next('/')
      return
    }
  }

  next()
})
```

**权限控制流程**：
1. 用户访问 `/admin/dashboard`
2. 路由守卫检查 `requiresAdmin` 标记
3. 验证用户是否登录
4. 检查用户是否有管理员权限（`is_admin`）
5. 非管理员用户被重定向到首页

## 🔧 技术难点与解决方案

### 难点1：管理员权限的验证

**问题**：如何确保只有管理员能访问后台功能？

**解决方案**：

**前端层面**：
```typescript
// 路由守卫检查
if (to.meta.requiresAdmin) {
  const user = authApi.getCurrentUser()
  if (!user || !user.is_admin) {
    next('/')  // 非管理员跳转到首页
    return
  }
}
```

**后端层面**：
```python
# 使用JWT验证
verify_jwt_in_request()

# 从Token中获取用户信息
user_id = get_jwt_identity()
user = User.query.get(user_id)

# 检查是否是管理员
if not user.is_admin:
    return jsonify({'message': '权限不足'}), 403
```

**双重验证**：前端路由守卫 + 后端JWT验证

### 难点2：统计数据的性能优化

**问题**：多个统计查询可能影响性能

**解决方案**：

```python
# 使用count()聚合查询，比查询所有数据快得多
```

**优化效果**：
- `count()` 只返回数量，不加载具体数据
- 数据库层面进行计数，减少内存使用
- 查询速度快，响应时间短

## 🎓 扩展知识点

### 1. RBAC（基于角色的访问控制）
```
普通用户：浏览菜谱、创建菜谱、评论
管理员：上述权限 + 用户管理 + 数据统计 + 内容审核
```

### 2. JWT认证流程
```
用户登录 → 后端生成Token → 前端保存Token
→ 后续请求携带Token → 后端验证Token → 返回数据
```

### 3. 数据库聚合查询
```python
# 使用COUNT聚合函数，不加载具体数据
count = db.session.query(Model).count()

# 比查询所有数据再统计快得多
# users = db.session.query(User).all()
# count = len(users)  # ❌ 低效
```
