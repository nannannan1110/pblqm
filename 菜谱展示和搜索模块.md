1. **菜谱列表展示**：以卡片形式展示所有菜谱

4. **排序功能**：支持按创建时间、标题、准备时间等排序
5. **菜谱详情查看**：点击菜谱卡片查看完整信息

### 4. 后端多条件搜索实现

**文件位置**：`backend/app/routes/recipes.py:8-74`

```python
@recipes_bp.route('/', methods=['GET'])
def get_recipes():
    # 获取查询参数
    page = request.args.get('page', 1, type=int)
    per_page = request.args.get('per_page', 10, type=int)
    search = request.args.get('search', '').strip()
    difficulty = request.args.get('difficulty', '').strip()
    max_prep_time = request.args.get('max_prep_time', type=int)
    max_cook_time = request.args.get('max_cook_time', type=int)
    sort_by = request.args.get('sort_by', 'created_at')
    sort_order = request.args.get('sort_order', 'desc')

    # 构建查询
    query = Recipe.query

    # 关键词搜索（多字段模糊匹配）
    if search:
        from sqlalchemy import or_
        search_filter = or_(
            Recipe.title.contains(search),      # 标题匹配
            Recipe.description.contains(search), # 描述匹配
            Recipe.ingredients.contains(search)  # 食材匹配
        )
        query = query.filter(search_filter)

    # 难度筛选
    if difficulty:
        query = query.filter(Recipe.difficulty == difficulty)

    # 准备时间筛选
    if max_prep_time:
        query = query.filter(Recipe.prep_time <= max_prep_time)

    # 烹饪时间筛选
    if max_cook_time:
        query = query.filter(Recipe.cook_time <= max_cook_time)

    # 动态排序
    if sort_by == 'created_at':
        order_field = Recipe.created_at.desc() if sort_order == 'desc' else Recipe.created_at.asc()
    elif sort_by == 'title':
        order_field = Recipe.title.desc() if sort_order == 'desc' else Recipe.title.asc()
    elif sort_by == 'prep_time':
        order_field = Recipe.prep_time.desc() if sort_order == 'desc' else Recipe.prep_time.asc()

    query = query.order_by(order_field)

    # 分页查询
    recipes = query.paginate(
        page=page,
        per_page=per_page,
        error_out=False
    )

    # 返回JSON响应
    return jsonify({
        'recipes': [recipe.to_dict() for recipe in recipes.items],
        'total': recipes.total,
        'pages': recipes.pages,
        'current_page': page
    })
```

**技术要点**：
- **SQLAlchemy ORM** 进行数据库查询
- **多字段模糊搜索**：使用 `or_()` 组合多个条件
- **动态排序**：根据不同字段和方向排序
- **分页查询**：使用 `paginate()` 方法
- **链式调用**：优雅的查询构建方式

```typescript
// 获取完整的图片URL
const getImageUrl = (image?: string) => {
  // 如果没有图片，返回默认图片
  if (!image) return defaultImage

  // 如果已经是完整URL，直接返回
  if (image.startsWith('http')) return image

  // 如果是相对路径，添加后端域名
  if (image.startsWith('/static')) return `http://localhost:5000${image}`

  // 如果是只有文件名，拼接完整路径
  return `http://localhost:5000/static/uploads/images/${image}`
}
```

**设计亮点**：
- 兼容多种图片URL格式
- 容错处理：无图片时显示默认图
- 环境解耦：相对路径便于部署

## 🔧 技术难点与解决方案

### 难点1：多字段模糊搜索的实现

**问题**：用户输入的关键词需要同时匹配标题、描述、食材三个字段

**解决方案**：
```python
from sqlalchemy import or_

search_filter = or_(
    Recipe.title.contains(search),
    Recipe.description.contains(search),
    Recipe.ingredients.contains(search)
)
query = query.filter(search_filter)
```

**技术点**：使用 SQLAlchemy 的 `or_()` 逻辑运算符

---

### 难点2：分页时保持搜索条件

**问题**：翻页时需要保持当前的搜索和筛选条件

**解决方案**：
```typescript
// 分离搜索表单和实际搜索参数
const searchForm = ref({ ... })  // 用户输入的表单
const searchParams = ref({ ... }) // 实际执行的搜索参数

// 点击搜索按钮时才更新搜索参数
const handleSearch = () => {
  searchParams.value = { ...searchForm.value }
  pagination.value.currentPage = 1  // 重置页码
  fetchRecipes()
}

// 翻页时使用保存的搜索参数
const fetchRecipes = async () => {
  const queryParams = {
    page: pagination.value.currentPage,
    per_page: pagination.value.pageSize,
    ...searchParams.value  // 使用保存的搜索参数
  }
  // ...
}
```

### 难点3：动态排序的实现

**问题**：用户可以选择不同的排序字段和方向

**解决方案**：
